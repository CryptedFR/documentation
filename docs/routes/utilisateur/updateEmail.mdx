# Modifier l'email

Cette route permet de modifier l'email de l'utilisateur authentifié.

## URL

<HttpMethod method="PATCH" route="/v1/user/email" />

## En-têtes de la requête

| Nom      |        Valeur ou type                  |
|-----------|------------------------------|
| `Content-Type`     | `application/json`            |
| `Authorization`     | `Bearer <token>`            |

## Corps de la requête

| Champ            | Type     | Requis | Description                           |
|------------------|----------|--------|---------------------------------------|
| `email`          | `string` | Oui    | Nouvelle adresse email de l'utilisateur.      |

### Exemple

```json
{
  "email": "john.doe123@gmail.com",
}
```

## Réponse

### Succès

- **Code HTTP** : `200 OK`

| Nom      |        Valeur ou type                  | Description                                         |
|-----------|------------------------------|-----------------------------------------------------|
| `status`     | `success`            | Status de la réponse.               |
| `statusCode`     | `200`          | Code HTTP de la réponse. |
| `code`     | `S_EMAIL_UPDATED`        | Code success.                |

- **Exemple JSON :**

```json
{
  "status": "success",
  "statusCode": 200,
  "code": "S_EMAIL_UPDATED"
}
```
