# Modifier le mot de passe

Cette route permet de modifier l'email de l'utilisateur authentifié.

## URL

<HttpMethod method="PATCH" route="/v1/user/password" />

## En-têtes de la requête

| Nom      |        Valeur ou type                  |
|-----------|------------------------------|
| `Content-Type`     | `application/json`            |
| `Authorization`     | `Bearer <token>`            |

## Corps de la requête

| Champ            | Type     | Requis | Description                           |
|------------------|----------|--------|---------------------------------------|
| `old_password`          | `string` | Oui    | Ancien mot de passe de l'utilisateur.      |
| `password`          | `string` | Oui    | Nouveau mot de passe de l'utilisateur.      |
| `password_confirmation`          | `string` | Oui    | Confirmation du nouveau mot de passe de l'utilisateur.      |

### Exemple

```json
{
  "old_password": "Johndoe0419*",
  "password": "J0hn2024$",
  "password_confirmation": "J0hn2024$",
}
```

## Réponse

### Succès

- **Code HTTP** : `200 OK`

| Nom      |        Valeur ou type                  | Description                                         |
|-----------|------------------------------|-----------------------------------------------------|
| `status`     | `success`            | Status de la réponse.               |
| `statusCode`     | `200`          | Code HTTP de la réponse. |
| `code`     | `S_PASSWORD_UPDATED`        | Code success.                |

- **Exemple JSON :**

```json
{
  "status": "success",
  "statusCode": 200,
  "code": "S_PASSWORD_UPDATED"
}
```

### Erreurs

- **Code HTTP** : `401 Unauthorized` ou `422 Unprocessable Entity`

| Nom      |        Valeur   ou type               | Description                                         |
|-----------|------------------------------|-----------------------------------------------------|
| `status`     | `error`            | Status de la réponse.               |
| `statusCode`     | `401` ou `422`        | Code HTTP de la réponse. |
| `code`     | `E_UNAUTHORIZED_ACCESS`  ou `E_INVALID_OLD_PASSWORD` ou `E_VALIDATION_ERROR`     | Code erreur qui permet d'identifier l'erreur                |
| `message`     | `Unauthorized access` ou `Old password is invalid`        | Message de l'erreur     |
| `errors`     | `Array`        | Erreurs de validation     |


- **Exemple JSON :**

```json
{
  "status": "error",
  "statusCode": 401,
  "code": "E_UNAUTHORIZED_ACCESS",
  "message": "Unauthorized access"
}

// ou

{
  "status": "error",
  "statusCode": 422,
  "code": "E_INVALID_OLD_PASSWORD",
  "message": "Old password is invalid"
}

// ou

{
  "status": "error",
  "statusCode": 422,
  "code": "E_VALIDATION_ERROR",
  "message": "Validation failure",
  "errors": [
    {
      "code": "confirmed",
      "field": "password"
    }
  ]
}
```
