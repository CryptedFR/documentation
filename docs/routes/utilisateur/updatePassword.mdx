# Modifier le mot de passe

Cette route permet de modifier l'email de l'utilisateur authentifié.

## URL

<HttpMethod method="PATCH" route="/v1/user/password" />

## En-têtes de la requête

| Nom      |        Valeur ou type                  |
|-----------|------------------------------|
| `Content-Type`     | `application/json`            |
| `Authorization`     | `Bearer <token>`            |

## Corps de la requête

| Champ            | Type     | Requis | Description                           |
|------------------|----------|--------|---------------------------------------|
| `old_password`          | `string` | Oui    | Ancien mot de passe de l'utilisateur.      |
| `password`          | `string` | Oui    | Nouveau mot de passe de l'utilisateur.      |
| `password_confirmation`          | `string` | Oui    | Confirmation du nouveau mot de passe de l'utilisateur.      |

### Exemple

```json
{
  "old_password": "Johndoe0419*",
  "password": "J0hn2024$",
  "password_confirmation": "J0hn2024$",
}
```

## Réponse

### Succès

- **Code HTTP** : `200 OK`

| Nom      |        Valeur ou type                  | Description                                         |
|-----------|------------------------------|-----------------------------------------------------|
| `status`     | `success`            | Status de la réponse.               |
| `statusCode`     | `200`          | Code HTTP de la réponse. |
| `code`     | `S_PASSWORD_UPDATED`        | Code success.                |

- **Exemple JSON :**

```json
{
  "status": "success",
  "statusCode": 200,
  "code": "S_PASSWORD_UPDATED"
}
```
